专业论文写作课堂笔记1

Oct 8, 2018 | 课堂笔记 | 阅读

时间：10月8日周一早上

课前参考阅读：

Two papers on clustering

  • BIRCH: An Efficient Data Clustering Method for Very Large Databases.

    SIGMOD Conference 1996: 103-114

  • CURE: An Efficient Clustering Algorithm for Large Databases.

    SIGMOD Conference 1998: 73-84

写论文时，不是从头写到尾，先挑重要部分开始写

Structures

  • Abstract
  • Introduction
  • Preliminaries（背景/前置知识）
  • Main part（你的贡献）
  • Conclusions(and discussion)
  • References

整篇论文最重要的地方是Abstract、Introduction和Conclusion，决定了是否被录用

区别：

​ Abstract 骗人看，Introduction和Conclusions骗人Accept。——李开复

  • Abstract夸自己的文章有多好，200-300个英语单词， 包含Attribution。

  • Introduction 是整片文章的缩写， 包含1. 大背景，小背景， 假设，范畴。2. 问题是什么，问题维度，难在哪里，问题价值。3. 基本思路，概述。4. 贡献点， Attribution。 5.接下来的文章组织。

    • 一般不去质疑作者假设和范围
    • 不能让读文章的人总结出难在哪里，写文章的人要自己提炼
    • 一般来说贡献点有三到五个之间
    • 假设别人是小白，一点不懂，逻辑上要很顺，把读者引进文章。
  • Conclusions可以提取得的成绩，用数据， 分析性的东西， 我取得这个成绩主要归功于我用了哪些方法， 拔高， future work。
    • 假设别人至少已经读过Introduction
    • discussion，换一个场景，别人还可以做什么future work

具体论文阅读

BIRCH

  • 文章题目规范要求
    • 专有名词缩写全部大写
    • 实词（名词动词形容词副词）首字母大写
    • 介词全部小写
    • 冠词在句首首字母大写（冒号后等于另起一行）An、The、A
    • 文章若有低级错误（语法，大小写，拼写）， 会被评审人直接否决
  • 作者，单位，email
  • Abstract
    • 一般会议和期刊论文不分段
    • 大背景—>小背景—>快速缩到自己要做的内容
    • 我提出的东西很好很好
    • 如果语法很好，特别有信心，可以写长难句，逗号前后主语一致等等，如果语法不好，最好写简单句。一篇文章可以有几句长难句，大部分都是简单句
    • 一页尽量控制只有一个语法错误，不要出错在关键句子
  • Preliminaries（前导知识）
    • Problem statement/description/definition
    • Related work

英文搭配

  1. address the problem
  2. this paper presents（提出）a data clustering method named BIRCH
  3. and demonstrates that it is espacially suitable for large databases
  4. large database and（递进） minimization of IO costs

作业

  1. 看五六篇论文
  2. 看算法
    1. b-tree
    2. 插入排序/快速排序/冒泡排序
    3. 图算法（最短路径）
  3. 如果要写论文，讲这些算法，你会怎么写，能不能更精炼。
